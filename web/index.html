shell-remote

远程协作终端

Agent 模式使用说明

快速开始

在目标机器上下载并运行 shell-remote 代理模式，它会连接到你的 relay 服务器。

$ ./shell-remote-{arch} agent --relay-url https://your-host

参数说明

  • --relay-url — relay 服务器地址（支持 ws:// / wss:// / https://）
  • --key — 固定密钥（可选，不指定则生成临时密钥）
  • --root — 文件管理器默认目录（默认 $HOME）
  • --token-type — 密钥类型（rw / ro / both，默认 rw）
  • --shell — Shell 路径（默认 /bin/bash）

使用固定密钥

固定密钥在 relay 重启后仍然有效，适合长期部署：

$ ./shell-remote-agent agent --relay-url https://your-host --key my-secret-key

只读模式

只读 token 只能查看终端输出和文件列表，不能执行命令或修改文件：

$ ./shell-remote-agent agent --relay-url https://your-host --token-type ro

双模式 (rw + ro)

同时生成读写和只读两个密钥：

$ ./shell-remote-agent agent --relay-url https://your-host --token-type both

启动后 relay 会打印会话密钥，将密钥分发给浏览器用户即可远程协作。 二进制文件兼容 Linux 内核 3.2+，无需任何运行时依赖。

下载二进制文件

二进制文件使用 musl 静态编译 — 无需运行时依赖，兼容任意 Linux 内核 3.2+。

也支持 curl 一键下载，点击卡片上的 copy 按钮复制命令。
